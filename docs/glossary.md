# GitHub Projects Glossary

Kamus istilah-istilah penting dalam GitHub Project Management.

## A

**Acceptance Criteria**
Kondisi yang harus dipenuhi agar sebuah issue dianggap selesai. Biasanya ditulis dalam bentuk checklist.

**Agile**
Metodologi pengembangan software yang iterative dan incremental, dengan fokus pada collaboration dan adaptability.

**Assignee**
Person yang bertanggung jawab mengerjakan sebuah issue atau pull request.

**Automation**
Fitur untuk automatically update status, fields, atau actions berdasarkan events tertentu.

## B

**Backlog**
Daftar work items yang belum dijadwalkan untuk dikerjakan. Biasanya sorted by priority.

**Board View**
Tampilan kanban-style dengan columns untuk visualize workflow.

**Blocker**
Issue atau dependency yang menghalangi progress dari task lain.

**Burndown Chart**
Chart yang menunjukkan remaining work vs time dalam sprint.

**Bug**
Software defect yang perlu diperbaiki.

## C

**Cycle Time**
Waktu dari saat work dimulai sampai selesai (In Progress â†’ Done).

**Custom Field**
Field tambahan yang bisa ditambahkan ke project items (Priority, Story Points, etc).

**CI/CD (Continuous Integration/Continuous Deployment)**
Practice untuk automatically test dan deploy code changes.

## D

**Daily Standup**
Daily meeting singkat (15 min) untuk sync team progress.

**Definition of Done (DoD)**
Checklist kriteria yang harus dipenuhi agar work dianggap complete.

**Dependencies**
Relationship antara tasks dimana satu task depends on another.

**Draft Issue**
Note atau idea yang belum menjadi formal issue di repository.

**DRI (Directly Responsible Individual)**
Satu person yang accountable untuk completion suatu task.

## E

**Epic**
Large body of work yang bisa dipecah menjadi smaller issues. Biasanya multi-sprint.

**Estimate**
Predicted effort atau time untuk complete suatu task (story points atau hours).

## F

**Feature**
New functionality yang ditambahkan ke product.

**Filter**
Cara untuk show subset of items based on criteria tertentu.

## G

**GitHub Actions**
Automation platform untuk CI/CD dan workflows di GitHub.

**Grooming (Backlog Refinement)**
Meeting untuk review dan prepare backlog items untuk future sprints.

## I

**Issue**
Unit of work di GitHub - bisa berupa bug, feature, task, etc.

**Iteration**
Fixed time period untuk complete set of work (sama dengan Sprint).

## K

**Kanban**
Visual workflow management method dengan WIP limits dan continuous flow.

**KPI (Key Performance Indicator)**
Metrics untuk measure success (velocity, cycle time, etc).

## L

**Label**
Tag untuk categorize issues (bug, feature, priority, etc).

**Lead Time**
Total waktu dari request sampai delivery (dari backlog ke production).

## M

**Milestone**
Significant event atau deliverable dalam project timeline.

**MVP (Minimum Viable Product)**
Version dengan minimum features untuk validate product idea.

## P

**Pull Request (PR)**
Request untuk merge code changes ke repository.

**Priority**
Importance level dari suatu issue (Critical, High, Medium, Low).

**Product Backlog**
Ordered list of everything yang mungkin needed di product.

## R

**Retrospective (Retro)**
Meeting di end of sprint untuk reflect dan improve.

**Roadmap View**
Timeline visualization showing schedule dan deadlines.

**Review**
Process untuk evaluate completed work (code review, sprint review).

## S

**Scrum**
Agile framework dengan fixed-length sprints dan specific ceremonies.

**Sprint**
Fixed time period (biasanya 2 weeks) untuk complete committed work.

**Sprint Planning**
Meeting untuk plan work di upcoming sprint.

**Sprint Review**
Meeting untuk demo completed work ke stakeholders.

**Story Points**
Unit of measure untuk relative effort/complexity suatu task.

**Stakeholder**
Person dengan interest dalam project outcome (users, management, etc).

## T

**Table View**
Spreadsheet-style view showing all items dan fields.

**Technical Debt**
Accumulated shortcuts atau workarounds yang perlu diperbaiki later.

**Throughput**
Number of items completed dalam time period tertentu.

**Ticket**
Another term untuk issue atau work item.

## V

**Velocity**
Average amount of work (story points) completed per sprint.

**View**
Different way to display project data (Board, Table, Roadmap).

## W

**WIP (Work In Progress)**
Items yang currently being worked on.

**WIP Limit**
Maximum number of items allowed dalam In Progress state.

**Workflow**
Sequence of steps yang item goes through dari start sampai done.

## Z

**Zero Bug Policy**
Practice untuk fix bugs immediately before starting new features.

### Status Terms

- **Todo**: Ready to start
- **In Progress**: Currently working
- **In Review**: Awaiting feedback
- **Blocked**: Can't proceed
- **Done**: Completed
- **Archived**: Removed from active view

### Priority Levels

- **P0/Critical**: Drop everything, fix now
- **P1/High**: Fix ASAP
- **P2/Medium**: Fix soon
- **P3/Low**: Fix when possible
- **P4/Trivial**: Nice to have

### Estimation Terms

- **T-shirt sizing**: XS, S, M, L, XL
- **Fibonacci**: 1, 2, 3, 5, 8, 13, 21
- **Hours**: Direct time estimate

### Meeting Types

- **Planning**: Plan sprint work
- **Standup**: Daily sync
- **Review**: Demo completed work
- **Retrospective**: Reflect and improve
- **Grooming**: Refine backlog
